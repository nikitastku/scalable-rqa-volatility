================================================================================
  F. KEY FINDINGS FOR RQ3 (non-stationarity stress test)
================================================================================

1. DEEP LEARNING DOMINATES: LSTM with RV features achieves AUC=0.975,
   far ahead of all other approaches. Adding lagged RV as input was the
   single most impactful improvement (+0.046 F1 for GINN at λ=0.3).

2. TEACHER DISTILLATION FAILS: GINN's GJR-GARCH teacher (AUC=0.888)
   is worse than the student. Distillation hurts monotonically:
   λ=0.0 (0.975) > λ=0.1 (0.974) > λ=0.3 (0.968) > λ=0.5 (0.954).

3. RQA FEATURES AT CHANCE: All RQA-only and β-RQA-only variants perform
   near random (AUC 0.48–0.56). The extreme distribution shift means
   recurrence patterns calibrated on high-volatility training data
   carry no discriminative signal in the calm test period.

4. RQA CONFIG INSENSITIVE: Varying window (30/60/90), embedding (m=3-6,
   tau=1-3), and mode (joint/per_series) made virtually no difference.
   RF Std+RQA hovers at AUC 0.877–0.879 regardless of config.

5. β SELECTION MATTERS SLIGHTLY: β=1.5 is best for RF Std+β-RQA
   (AUC=0.874), but all β values underperform RF Std alone (0.879).
   The horizontal line measures (LAM_h, TT_h) provide modest benefit
   for β-RQA-only (F1 improved vs old 6-feature version).

6. STANDARD FEATURES SUFFICIENT: Simple rolling statistics (ret_abs,
   ret_sq, rv, rolling means/stds) with RF capture nearly all
   predictable signal on this dataset (AUC=0.879).

CONCLUSION FOR DATASET 1: Under extreme non-stationarity, recurrence-
based features add no value. The real test of RQA/β-RQA will be
Dataset 2, where train and test share similar distributional properties.

1. LSTM REMAINS KING (AUC=0.993): Deep learning with RV features dominates,
   though on a smaller effective test set (86 rows due to seq_len trimming).

2. HAR-RV IS REMARKABLY STRONG (AUC=0.986, F1=0.917): Simple linear model

3. ★ RQA FEATURES NOW ADD VALUE ★ — The headline result:
   - LogReg Std+RQA (per_series) achieves F1=0.934, AUC=0.972
     This BEATS RF Standard alone (F1=0.917) on F1.
   - RF Std+β-RQA (β=0.5) achieves F1=0.904, AUC=0.974
   - Multiple combined models exceed RF Std (AUC=0.973) in AUC.
   This is the OPPOSITE of Dataset 1 where RQA always hurt.

4. ★★ BEYOND-LITERATURE β VALUES EXCEL ★★ — Novel finding:
   - RF Std+β-RQA at β=4.0 achieves F1=0.938, AUC=0.978
     This is the BEST RF F1 across all feature baselines.
   - LogReg Std+β-RQA at β=5.0 achieves AUC=0.987
     This SURPASSES HAR-RV (0.986) and approaches LSTM (0.993).
   - Deckert et al. (2025) tested up to β=4; β=5 is genuinely novel.
   The optimal β for financial volatility data lies ABOVE the standard
   literature range, suggesting large-value emphasis is beneficial.

5. PER-SERIES RQA MODE WINS FOR LOGREG: Separating RQA features for
   log_return and rv (12 features) rather than joint embedding (6 features)
   enables LogReg to achieve the best F1 of any feature baseline.

6. EMBEDDING PARAMETERS MATTER: m=3 (not default m=4) is best for LogReg
   with β-RQA (AUC=0.984). Higher m degrades performance. tau=2 is a
   robust default for RF; LogReg can benefit from tau=4.

7. TEACHER DISTILLATION STILL HURTS: Same monotonic degradation as
   Dataset 1 — the GJR-GARCH teacher (AUC=0.829) is too weak.

8. RQA-ONLY STILL WEAK BUT IMPROVING: Best standalone is LogReg b-RQA-only
   at b=5.0 (AUC=0.786), up from 0.742 at b=0.5. Recurrence features are
   complementary to standard features, not standalone replacements.

9. STATISTICAL SIGNIFICANCE CAVEAT: With only 338 test rows, bootstrap
   confidence intervals are wide and no pairwise comparison reaches
   statistical significance (see Section F2). Point estimates suggest
   improvement, but the evidence is inconclusive at this sample size.
   Dataset 3 (450K test rows) provides the statistically rigorous evidence.

1. STANDARD FEATURES DOMINATE AT SCALE: LogReg and RF with 18 precomputed

   LogReg Std+RQA and LogReg Std+b-RQA show negligible per-stock
   differences (p = 0.94 and p = 0.23 respectively). The standard
   features already saturate LogReg's capacity on this dataset.

5. RQA CONFIG MATTERS LITTLE AT SCALE: All window sizes and modes produce
   nearly identical combined results. The cross-sectional pooling across
   503 stocks overwhelms per-window differences.

6. b=2.0 IS BEST FOR STANDALONE b-RQA: LogReg b-RQA-only at b=2.0
   achieves AUC=0.672, the best standalone recurrence result.

7. LSTM/GINN LIMITED BY SUBSAMPLE: 150 tickers used due to memory.
   AUC=0.914 on 125K rows. Not directly comparable to 503-stock baselines.

8. TEACHER DISTILLATION STILL HURTS: Same monotonic pattern as D1/D2.

9. SCALABILITY DEMONSTRATED (RQ2): RQA computed on 3M rows across 503
   stocks in 316-624 seconds with step=20. This validates the
   "scalable" claim in the thesis title.

cross-sectional diversity overwhelms recurrence feature differences.
However, standalone β-RQA is sensitive to m (m=5 best, AUC=0.772).

  RQ4: ALTERNATIVE APPROACHES FOR SCALABLE RECURRENCE PLOT REPRESENTATIONS
  Literature review + published benchmark comparison
================================================================================

Research Question 4: What alternative approaches for scalable recurrence plot
representations exist in the literature, and how could such approaches be
adapted or extended to improve computational efficiency for large financial
time series?

================================================================================
  A. ALTERNATIVE APPROACHES IN THE LITERATURE
================================================================================

Six main approaches to scalable RQA computation have been identified:

1. RQA_woRP — Direct RQA without Recurrence Plot 
   Computes line length histograms directly from phase space vectors without
   constructing the full N x N recurrence matrix. Reduces memory from O(N^2)
   to O(N) and computation steps by >50%, while producing EXACT results.
   At N=25,000: 0.48s vs 2.06s for standard RQA (4.3x speedup).

2. RQA_Samp — Sampled RQA 
   Extends RQA_woRP with random sampling: instead of testing all N^2 pairs,
   randomly samples M pairs and measures line lengths from valid starts.
   With M=4N: 0.26s at N=25,000 (7.9x speedup, near-exact results).
   With M=0.2N: 0.013s at N=25,000 (158x speedup, DET deviation <10^-4).
   Accuracy: M=500 samples already yields DET deviations <10^-4 for N=25,000.

3. Recurrence Microstates 
   Approximates DET and LAM from sampled 3x3 sub-matrices of the RP.
   At N=25,000 with M=4N: 0.018s (114x speedup over standard RQA).
   Trade-off: less accurate than RQA_Samp for the same M, but faster
   due to highly optimised compiled C code in the Julia implementation.

4. RQA_OpenMP — Parallel CPU RQA 
   C-based implementation using OpenMP multithreading for parallel RP
   construction and RQA computation. Exact results, O(N^2/P) where P
   is the number of threads.
   At N=200,000 with 128 threads: 1 min 51s vs 9 min 17s single-thread.

5. PyRQA — OpenCL GPU-accelerated RQA 
   Massively parallel RQA using OpenCL framework on GPUs. Processes
   time series of >1 million data points. Uses divide-and-recombine
   paradigm to split the RP across GPU devices.
   At N=200,000 with 2x GPU: 11.4s vs 9 min 17s single-thread (49x).
   At N=1,000,000: ~5 min with 2x GPU (Potsdam climate data).

6. AccRQA — Multi-platform accelerated RQA 
   High-performance CPU and NVIDIA GPU implementation. Requires
   CMake and C++ compiler (or CUDA for GPU mode).
   Benchmarked on HPC with Nvidia H100 GPU in rp_speedtest repo.

================================================================================
  D. OUR APPROACH: WINDOWED RQA WITH STRIDE
================================================================================

Our implementation uses a windowed approach with stride, which is conceptually
related to the sampling idea of RQA_Samp but operates at the window level:

  - Instead of computing RQA on the full N-length series (O(N^2)),
    we compute RQA on windows of size w with stride s.
  - Effective operations: (N/s) windows x O(w^2) per window = O(N*w^2/s)
  - For our D3 config: w=60, s=20, so each window is a 60x60 RP
    (only 3,600 comparisons vs N^2 for the full series).

Our measured computation times on Dataset 3 (3M rows, 503 stocks):
  RQA (w=60, s=20):          316s   (~5.3 min)
  RQA (w=120, s=20):         597s   (~10 min)
  beta-RQA (w=60, s=20):     ~450s  (~7.5 min)
  Preprocessing pipeline:    75s    (feature computation for all stocks)

Per-stock average: 316s / 503 stocks = 0.63s per stock (~6,000 bars each)
Per-window average: 0.63s / (6000/20) = 0.002s per window (60-bar window)

================================================================================
  E. COMPARISON: OUR APPROACH vs PUBLISHED BENCHMARKS
================================================================================

For a single time series of N=6,000 (typical per-stock length in D3):

  Approach                    Estimated time    Notes
  ────────────────────────    ──────────────    ─────
  Standard RQA (full RP)      ~0.12s*          O(N^2), extrapolated
  RQA_woRP                    ~0.03s*          4.3x faster than standard
  RQA_Samp (M=4N)             ~0.015s*         7.9x faster
  Our windowed (w=60,s=20)    ~0.63s           300 windows, each tiny
  PyRQA (OpenCL, 1x GPU)      ~0.03s*          GPU-accelerated

  * Extrapolated from N=25,000 benchmarks using O(N^2) scaling

Our windowed approach is SLOWER than a single full-series RQA for N=6,000
because it computes 300 separate small RQAs. However, it produces a TIME
SERIES of RQA features (one per window), which is essential for our ML
pipeline — a single full-series RQA would give only one feature vector per
stock, losing all temporal dynamics.

The windowed approach is thus not an alternative to full-series RQA, but a
different computation mode designed for feature extraction in ML pipelines.

================================================================================
  F. ADAPTATION AND EXTENSION PROPOSALS FOR FINANCIAL TIME SERIES
================================================================================

1. COMBINE RQA_Samp WITH beta-DIVERGENCE (Novel proposal)
   Marwan's sampling approach samples random (i,j) pairs and measures
   line lengths. This could be extended to use beta-divergence distance
   (Deckert et al. (2025)) instead of Euclidean distance for the recurrence condition.
   Expected benefit: Faster computation (from sampling) + better sensitivity
   to volatility regime transitions (from beta-divergence).
   Neither paper currently combines these two ideas.

2. USE PyRQA/AccRQA AS BACKEND FOR WINDOWED COMPUTATION
   Replace our per-window scipy.spatial.distance.cdist + numpy RQA with
   PyRQA's OpenCL-accelerated computation for each window. Since PyRQA
   handles the full RP construction + RQA in one call, this would avoid
   the Python-level overhead of our implementation.
   Expected benefit: 5-10x speedup per window on GPU-equipped machines.

3. ADAPTIVE WINDOW SIZING BASED ON VOLATILITY
   Use smaller windows during calm periods (where recurrence structure is
   simple) and larger windows during volatile periods (where longer patterns
   emerge). This reduces computation in the majority of calm bars while
   preserving detail during regime transitions.
   Motivation: Financial data is inherently heterogeneous — most of the time
   series is low-volatility, with brief high-volatility episodes.

4. PARALLEL PER-STOCK COMPUTATION
   Our current implementation processes stocks sequentially. Using Python
   multiprocessing or joblib, RQA for each stock could be computed in
   parallel across CPU cores. With 8 cores and 503 stocks, this would
   reduce total time by ~8x (from ~5 min to ~40 seconds).

5. MICROSTATE-BASED FEATURES FOR ML (inspired by Cruz 2025)
   Instead of computing standard RQA measures from the full RP, compute
   the 3x3 microstate distribution directly and use the microstate
   probabilities as ML features. This provides a richer feature set
   (up to 2^9 = 512 microstate types) and is naturally fast through
   the sampling approach.

================================================================================
  G. SUMMARY FOR RQ4
================================================================================

The literature offers several approaches to scalable RQA, ranging from
algorithmic improvements (RQA_woRP: 4x, RQA_Samp: up to 158x speedup)
to hardware acceleration (PyRQA/AccRQA: up to 49x on GPU). Our windowed
approach achieves scalability through a different mechanism — computing many
small RQAs rather than one large one — which is uniquely suited to ML
feature extraction where temporal resolution is needed.

For financial time series specifically, the most promising extensions would be:
(1) combining sampling with beta-divergence for faster yet more sensitive
    recurrence analysis, and
(2) parallelising per-stock computation for immediate practical speedup.

These approaches are complementary: algorithmic improvements reduce per-window
cost, hardware acceleration reduces per-computation cost, and our windowed
stride approach reduces the number of computations needed.
